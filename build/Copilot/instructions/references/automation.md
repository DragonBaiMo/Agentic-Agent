# 自动化开发规范

> 本文件是领域特化补充。通用编码规范见主文件 `code-quality.instructions.md`。

---

## 通用铁律

### 幂等性

- 重复执行不产生副作用
- 创建前检查是否存在
- 破坏性操作需 `-DryRun` 或二次确认

### 退出码

| 码 | 含义 |
|----|------|
| 0 | 成功 |
| 1 | 一般错误 |
| 2 | 参数错误 |

### 工具契约

- 工具 schema 必须显式定义
- 输入/输出类型明确
- 禁止猜工具字段

---

## 脚本开发

### 参数处理

- 显式参数契约，禁止位置歧义
- 输入校验：类型 · 范围 · 路径合法性
- 路径处理：防遍历 · 正确转义

### 输出规范

- 正常 → stdout · 错误 → stderr
- 结构化输出优于纯文本（JSON / CSV）
- 进度信息可读 · 可解析

### 常见陷阱

- 路径空格未转义
- 变量未加引号
- 错误未捕获导致静默失败
- 临时文件未清理

### 测试

- 幂等性测试：连续两次执行结果一致
- 边界测试：空输入 · 超大输入 · 特殊字符
- 失败测试：模拟各种失败场景

---

## 工作流编排

### 状态机

- 明确定义：pending / running / success / failed / cancelled
- 状态转换合法性校验
- 支持暂停/恢复/取消

### 任务依赖

- DAG（有向无环图）表达依赖
- 无依赖任务并发执行
- 失败策略：fail-fast / continue / retry

### 检查点

- 长任务必须有检查点
- 支持从检查点恢复
- 检查点数据持久化

### 防范

- 死锁：循环依赖
- 饥饿：低优先级任务永不执行
- 资源泄漏：任务失败未释放资源

### 测试

- 正常流程 · 失败恢复 · 并发 · 超时

## Agent 开发

### 工具设计原则

- 单一职责：一个工具做一件事
- 原子性：成功或失败，无中间态
- 可组合：工具间可自由组合

### 可靠性

| 机制 | 说明 |
|------|------|
| 重试 | 指数退避，有最大次数 |
| 超时 | 每个工具调用有超时 |
| 幂等 | 重复调用结果一致 |
| 去重 | 防止重复提交 |
| 补偿 | 失败时回滚/补偿 |

### 上下文管理

- 单 Agent：保持统一上下文
- 多 Agent：仅在任务可并行且可验证时启用
- 过长时：切分会话，关键信息落文件

### 防范

- 无限循环：Agent 陷入重复调用
