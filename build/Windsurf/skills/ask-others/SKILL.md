# 外部调研请求生成器 (Ask Others)

## 目的

在任务执行中遇到不确定性时，生成结构化调研请求文件供用户拿给外部 AI 调研。

## 触发流程

```
检测到不确定性
    ↓
生成调研请求文件 → 输出到当前工作目录
    ↓
向用户展示文件路径和摘要
    ↓
【暂停】等待用户拿去调研并返回结果
    ↓
用户提供调研结果后继续任务
```

## 输出文件规范

**文件名**：`ASK-OTHERS-{简短主题}.md`

**位置路径**：当前工作目录`.\askquestions\···`

**文件结构**：

```md
## 调研主题
我正在进行 {任务描述}，遇到以下技术问题：[一句话描述核心问题]

## 上下文背景
【背景】{上下文背景} [当前任务、为什么需要这个信息、已知相关信息]

## 相关伪代码

    ```pseudo
    【当前代码逻辑】[简化代码逻辑，展示问题所在位置]
    [用 // ← 不确定 标注关键位置]

    ```

## 技术约束
【技术环境】
- 技术栈：[语言/框架/版本]（如有）
- 运行环境：[OS/平台/依赖]（如有）
- 已有限制：[必须遵守的约束]（如有）

## 待澄清问题
1. [具体问题 1]
2. [具体问题 2]
{待澄清问题列表}

## 期望输出
请提供 {期望输出}。如有多种方案，请说明优缺点和适用场景:[代码示例/配置方式/最佳实践/对比分析等]

```

## 执行指令

1. **生成文件**：按上述格式在当前工作目录创建文件
2. **通知用户**：
   ```
   ⚠️ 检测到不确定性，已生成调研请求：

   文件：{文件路径}
   主题：{调研主题}

   请将文件内容（或底部调研 Prompt）提供给外部 AI 调研，
   然后将结果粘贴回来继续任务。
   ```
3. **等待响应**：暂停当前任务，直到用户提供研究结果
4. **继续执行**：收到调研结果后整合信息继续
