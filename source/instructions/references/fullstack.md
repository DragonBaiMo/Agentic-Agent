# 全栈开发规范

> 本文件是领域特化补充。通用编码规范（命名/类型安全/注释/错误处理/验证流程/安全基线）见主文件 `code-quality.instructions.md`。

---

## 前端

### 组件设计

- 单一职责：一个组件只做一件事
- 数据流单向：Props↓ Events↑
- 容器/展示分离：逻辑组件与 UI 组件分离

### 状态管理

- **本地状态**：组件内部 UI 状态
- **共享状态**：跨组件业务状态
- **服务端状态**：API 数据（SWR / React Query / TanStack Query）

原则：就近 · 不冗余 · 服务端与客户端分离

### 渲染性能

- 避免无谓重渲染（memo / useMemo / computed）
- 列表必须有稳定 key
- 大列表虚拟滚动
- 图片懒加载

### 可访问性（a11y）

- 语义化 HTML
- 表单标签关联
- 图片 alt 属性
- 键盘可访问
- 颜色对比度 WCAG AA（4.5:1）

### 性能优化

- 代码分割（路由级 · 组件级）
- 关键 CSS 内联
- 防抖/节流高频事件
- 静态资源强缓存 + 内容哈希

## 后端

### 分层架构

```
Controller（参数校验/路由）→ Service（业务/事务）→ Repository（数据访问）
```

### 依赖注入

- 构造函数/接口注入，便于测试替换

### 数据校验层级

```
Controller DTO/Schema 校验（必须）
    ↓
Service 业务规则校验（必须）
    ↓
数据库约束（最后防线）
```

### 性能优化

- 索引优化 · 避免 N+1
- 热点数据缓存
- 耗时操作异步化
- 连接池管理

### 部署清单

- [ ] 环境变量分离
- [ ] HTTPS 证书
- [ ] CORS 严格配置（禁止 `*`）
- [ ] 限流配置
- [ ] 健康检查端点
- [ ] 优雅关闭

## API 设计补充

### 认证方案选型

| 方案 | 适用场景 |
|------|----------|
| Session | 传统 Web · 同域 |
| JWT | 前后端分离 · 跨域 · 移动端 |
| OAuth 2.0 | 第三方登录 · 开放平台 |

### 安全防护速查

| 威胁 | 防护 |
|------|------|
| XSS | 输出编码 · CSP · HttpOnly Cookie |
| CSRF | SameSite Cookie · CSRF Token |
| SQL/NoSQL 注入 | 参数化查询 · ORM |
| SSRF | URL 白名单 · 禁止内网访问 |

## 测试策略

### 前端测试

| 类型 | 覆盖 |
|------|------|
| 单元测试 | 工具函数 · Hooks |
| 组件测试 | 组件交互 |
| E2E 测试 | 关键用户流程 |

### 后端测试

| 类型 | 覆盖 |
|------|------|
| 单元测试 | Service 业务逻辑 |
| 集成测试 | API 端点 |
| 契约测试 | API 契约一致性 |

### 测试原则

- 测试行为而非实现
- 关键路径必须覆盖
- 边界条件必须测试
- Mock 外部依赖
